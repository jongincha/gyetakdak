# 타이포그래피

## 스케일
- 폰트 크기는 일관된 스케일을 사용한다 (예: 12/14/16/18/20/24/32/40px, 또는 1.25배수 조화 스케일).
- 본문 기본 크기는 16px 이상을 권장한다. 14px 미만은 캡션/보조 텍스트 등 제한적인 용도로만 사용한다.
- 한 화면에 폰트 크기 종류가 5~6개를 넘지 않게 한다. 종류가 많아질수록 위계가 흐려진다.

## 위계 (Hierarchy)
- 제목/부제/본문/캡션처럼 명확한 텍스트 레벨을 정의하고, 크기·굵기·색상 중 최소 하나 이상을 다르게 해서
  위계를 시각적으로 구분한다.
- 굵기(font-weight)만으로 위계를 만들 때는 최소 2단계 차이(예: 400 vs 700)를 준다. 500 vs 600처럼
  미세한 차이는 화면에서 거의 구분되지 않는다.

## 줄간격 & 줄길이
- 본문 줄간격(line-height)은 1.4~1.6을 기본으로 한다. 제목은 1.1~1.3처럼 더 촘촘해도 된다.
- 한 줄의 글자 수는 약 45~75자(한글은 대략 30~45자) 범위를 넘지 않게 컨테이너 너비를 제한한다.
  너무 길면 눈이 다음 줄을 찾기 어렵다.

## 텍스트 처리
- 사용자 생성 콘텐츠(이름, 제목 등)는 길이가 예측 불가능하다는 전제로 설계한다: 줄바꿈, 말줄임(`ellipsis`),
  `word-break`을 명시적으로 처리하고 "짧은 텍스트"로만 테스트하지 않는다.
- 숫자 데이터(가격, 통계)는 등폭(tabular) 숫자를 사용하면 정렬된 표에서 자릿수가 흔들리지 않는다.
- 강조가 필요하면 색상보다 굵기/크기를 우선 사용한다. 색상 강조는 다크모드/색맹 사용자에게 덜 안정적이다.

## 흔한 실수
- 제목과 본문 폰트가 시스템 폰트 폴백에 따라 다르게 렌더링되는 걸 확인하지 않음.
- 버튼/라벨 텍스트가 길어질 때 컨테이너가 깨지는 걸 짧은 텍스트로만 테스트해서 놓침.
- `line-height: normal` 상태로 두어 폰트마다 줄간격이 들쭉날쭉함.
