# 색상 & 대비

## 팔레트 구성
- 브랜드/포인트 컬러 1개, 중립(그레이) 스케일 1세트, 시맨틱 컬러(success/warning/error/info) 세트로 구성한다.
- 그레이 스케일은 순수 흑백(`#000`, `#fff`)이 아니라 아주 약간 색조가 섞인 뉴트럴을 쓰면 더 고급스럽고
  다크모드 전환도 자연스럽다.
- 팔레트에 없는 색을 컴포넌트 하나를 위해 즉흥적으로 만들지 않는다. 새 색이 꼭 필요하면 기존 스케일에
  맞는 명도 단계로 추가하고, 재사용 가능하게 토큰화한다.

## 명도 대비 (WCAG)
- 본문 텍스트 대 배경: 최소 4.5:1 (AA 기준).
- 큰 텍스트(18px bold 이상 또는 24px 이상 일반) 대 배경: 최소 3:1.
- 아이콘, 테두리 등 UI 컴포넌트의 의미 전달용 요소: 최소 3:1.
- 회색 텍스트를 회색 배경 위에 올리는 조합(placeholder, disabled, secondary text)은 특히 대비가
  낮아지기 쉬우므로 반드시 실측 확인한다.

## 색만으로 정보를 전달하지 않기
- "빨간색 = 에러"만으로 표시하지 말고, 아이콘이나 텍스트 라벨을 함께 제공한다 (색맹 사용자 고려).
- 차트/그래프에서 카테고리 구분은 색상 외에 패턴, 라벨, 범례를 병행한다.

## 다크 모드
- 라이트 모드 색을 그대로 어둡게 반전하지 않는다. 다크 모드에서는 채도를 살짝 낮추고 명도를 조정해야
  눈이 편하다 (순수 흰색 텍스트 on 순수 검정 배경은 눈부심을 유발한다).
- 그림자(box-shadow)는 다크 모드에서 잘 보이지 않으므로, 테두리나 배경색 단계 차이로 위계를 표현하는
  것을 함께 고려한다.
- 이미지/일러스트에 흰 배경이 하드코딩되어 있으면 다크 모드에서 흰 박스로 튀어 보인다. 투명 배경을 쓰거나
  다크 모드 전용 에셋을 준비한다.

## 흔한 실수
- 브랜드 컬러를 본문 텍스트 색으로 그대로 사용 → 대비 부족으로 가독성 저하.
- hover/active 상태에서 배경색만 바꾸고 대비를 재확인하지 않음.
- 다크 모드를 라이트 모드 CSS에 `filter: invert()`로 때움 → 이미지/브랜드 컬러까지 반전되어 깨짐.
