# 레이아웃 & 간격

## 간격 스케일
- 4px 또는 8px 배수 스케일을 사용한다 (예: 4, 8, 12, 16, 24, 32, 48, 64).
  임의의 값(예: `13px`, `27px`)을 새로 만들지 않는다.
- 컴포넌트 내부 간격(padding)과 컴포넌트 간 간격(margin/gap)을 구분해서 스케일을 적용한다.
  내부는 촘촘하게(4~16px), 섹션 간은 넓게(32~64px) 가는 것이 기본값이다.
- 같은 위계의 요소들은 같은 간격을 공유한다. 카드 리스트라면 카드 사이 간격이 모두 동일해야 한다.

## 그리드 & 정렬
- 페이지 콘텐츠는 최대 너비(예: 1200~1440px)를 두고 중앙 정렬하는 것을 기본으로 한다. 초광폭 모니터에서
  텍스트 줄 길이가 지나치게 길어지지 않게 한다.
- 관련 요소는 시각적으로도 그룹으로 보이게 배치한다 (근접성의 법칙). 관련 없는 요소를 붙여놓지 않는다.
- 텍스트 정렬은 기본적으로 왼쪽(LTR 기준) 정렬. 가운데 정렬은 짧은 제목/배지 등 예외적인 경우에만 사용한다.
- 아이콘과 텍스트를 나란히 둘 때는 `align-items: center`로 광학적 중심을 맞춘다. 아이콘 자체의 여백 때문에
  시각적으로 어긋나 보이면 아이콘 쪽 패딩을 미세 조정한다.

## 컴포넌트 배치
- 행동 유도 버튼(CTA)은 사용자의 시선/스캔 흐름 끝(보통 우측 하단 또는 폼 맨 아래)에 배치한다.
- 파괴적 액션(삭제, 초기화)은 기본 액션과 시각적으로 분리하고, 실수로 나란히 눌리지 않게 간격을 둔다.
- 리스트/테이블은 스크롤이 필요한 경우 헤더를 고정(sticky)하는 것을 고려한다. 단, 화면이 항상 짧다면
  불필요한 sticky는 넣지 않는다.

## 흔한 실수
- 컴포넌트마다 다른 여백 값을 즉흥적으로 지정 → 화면 전체가 미묘하게 들쭉날쭉해 보임.
- 콘텐츠 길이에 따라 컨테이너 높이가 고정되어 텍스트가 잘림 → `min-height`를 쓰고 내용에 맞게 늘어나게 한다.
- 여백을 늘려서 "고급스러워 보이게" 하려다 화면 밀도가 낮아져 스크롤이 과도하게 길어짐 → 정보 밀도와
  여백의 균형을 콘텐츠 성격(대시보드 vs 랜딩페이지)에 맞게 조정한다.
